Admin Panel

Owner: Thilina Deepal

Logs

The "Logs" application provides a transparent view of error logs that occur within the system. With this app, users can easily access and review a comprehensive record of error occurrences. This allows for quick identification and troubleshooting of issues, enabling efficient maintenance and improvement of system performance. The "Logs" app serves as a valuable tool for system administrators and developers, offering insights into errors and facilitating a proactive approach to system management and problem resolution.

Apps

The "Apps" application is designed to streamline the management of applications within your system. With this app, you gain an overview of all installed applications, enabling you to update, remove, and add apps either from the DoCloud platform or by uploading app files in zip format directly to your system. Additionally, the app provides a convenient feature to reinitialize applications as needed. Furthermore, users can export their installed apps, ensuring a seamless experience for managing and maintaining the application ecosystem within the system.

Heartbeat/Scheduler

The "Heartbeat" or “Scheduler” app serves as a scheduling tool, triggering periodic cron jobs within the system. With this application, users can efficiently schedule and automate tasks, ensuring that essential processes occur at specified intervals. The "Heartbeat" app is particularly useful for automating routine activities, maintaining system health, and orchestrating timely actions. By providing a straightforward way to set up and manage cron jobs, this app contributes to the overall efficiency and reliability of scheduled tasks within the system.

Roles and Permissions

The "Roles and Permissions" module serves as a central hub for managing user roles and permissions within the system. In this module, administrators can seamlessly add new roles, define specific permissions for each role, and efficiently manage the overall permission structure. Additionally, the system allows for the adjustment of role hierarchy, ensuring a flexible and customized approach to user access and responsibilities. With this module, organizations can establish a robust framework for controlling user access, maintaining security, and tailoring roles to suit the specific needs and structure of the system.

Backups

In this section, users have the capability to perform comprehensive system backup and restoration operations. The functionality includes full backup and restore procedures, allowing users to safeguard their system data and configurations. Additionally, users can manage and delete existing backups as needed.

💡 It's important to note that restoring the database might result in data loss if the backup contains outdated information. It is advised to avoid performing write operations during the restoration period.

Settings

The "Settings" section serves as the central hub for configuring DoFramework settings, allowing users to customize various aspects of the framework according to their requirements. Within this module, users can adjust settings related to API configurations, database configurations, storage options, themes, and more. It provides a convenient and organized space for users to tailor the DoFramework to meet specific needs, ensuring flexibility and adaptability. Whether it's fine-tuning API integrations, optimizing database connections, adjusting storage preferences, or changing the overall theme, the "Settings" section is the go-to place for managing and customizing the DoFramework environment.
